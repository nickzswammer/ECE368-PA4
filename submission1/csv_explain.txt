This contains the feedback for your PA4 submission.

Warning messages from the compiler, if there are, are included in the
CE_log.txt.

The file grade_log.csv should be viewed as a spreadsheet.

There are three sections: one for the -b option, one for the -e option, and one for the main function.

pa4 -b:

The first column contains the names of the test cases.  The remaining 
columns are as follows: 

Compile error:
        0: if the executable was generated successfully
        1: if there were compilation errors

Time limit exceeded:
        -1: if your program crashed or did not complete
        0: if your program completed its execution within the time limit
        1: otherwise

Time(sec): the runtime of your program if it is a positive number

Valid tree:
	1: if your output file allows a binary tree to be constructed
		correctly
	0: if a tree can be built but something wrong is encountered
		i.e., the file has a wrong format
	-1: the file cannot be opened.


Contain all nodes:
	1: if the keys in the binary search tree match the final set
		of keys that remains after performing all operations_in_file
	0: otherwise

Valid BST:
	1: if the tree is a binary search tree (which may not contain
		all the necessary keys, checked by second integer)
	0: otherwise

Balanced:
	1: if the tree is height balanced
	0: otherwise

Fully match:
	1: if the output file matches the expected one
	0: otherwise

Used memory (bytes):
	amount of memory used by your program
	-1: when your program timed out, seg fault, or stack overflow

Stack overflow:
	0: if no overflow
	non-zero (timed out, segmentation fault, or actual overflow)

Memory leak:
	0: if no memory leak
	non-zero (timed out, segmentation fault, or actual memory leak)

Memory error:
	0: if no memory error
	non-zero (timed out, segmentation fault, or actual memory error)
